# 19. Questions to Confirm with Client

Trước khi code final hoặc trước khi báo giá/chốt phạm vi, nên hỏi khách các câu sau.

## 1. Scope

```txt
Giai đoạn 1 chỉ cần 3 landing page theo PDF hay cần chức năng bán hàng thật?
```

Nếu cần bán hàng thật, hỏi tiếp:

```txt
Có cần giỏ hàng, đặt hàng, thanh toán online, quản lý đơn hàng không?
```

## 2. Product Detail

```txt
Nút Coming soon có cần click được không?
```

```txt
Có cần trang chi tiết cho từng charm không?
```

```txt
Có cần hiển thị giá sản phẩm sau này không?
```

## 3. Brand Name

```txt
Tên chính thức trên website là YOUniverse hay UNIverse?
```

## 4. Language

```txt
Website dùng tiếng Anh, tiếng Việt hay song ngữ?
```

## 5. Assets

```txt
Khi nào có logo, banner và ảnh sản phẩm?
```

```txt
Banner là ảnh có sẵn text hay cần website code text đè lên ảnh?
```

```txt
Có yêu cầu kích thước/crop ảnh sản phẩm không?
```

## 6. Font

```txt
Font YOUTH và Montserrat có được phép nhúng vào website không?
```

```txt
Nếu YOUTH lỗi tiếng Việt có dấu, có đồng ý dùng Montserrat cho heading tiếng Việt không?
```

## 7. Animation

```txt
Khách muốn animation nhiều hay nhẹ?
```

```txt
Có thích hiệu ứng sparkle/star chuyển động không?
```

## 8. Footer

```txt
Thông tin contact trong footer đã chính xác chưa?
```

```txt
Social links đã là link chính thức chưa?
```

## 9. Hosting

```txt
Khách đã có domain/hosting chưa?
```

```txt
Muốn deploy lên Vercel không?
```

## 10. Maintenance

```txt
Sau này khách có muốn tự chỉnh sản phẩm/content không?
```

Nếu có, cần admin dashboard hoặc CMS.

## 11. Timeline

```txt
Deadline mong muốn là khi nào?
```

```txt
Có cần chia milestone không?
```

Suggested milestones:

1. UI draft.
2. Responsive.
3. Animation.
4. Asset replacement.
5. Final review.
6. Deploy.

## 12. Approval

```txt
Sau khi code xong, khách duyệt theo link preview hay gửi source code?
```

## Notes for Developer

Nếu khách chưa trả lời, chỉ nên code Phase 1 landing page. Không nên tự ý thêm e-commerce vì sẽ làm tăng scope và thời gian.
